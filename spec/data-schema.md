**Schema Documentation for `metropolitano_semifinal_clean.json`**

**File Purpose**  
This JSON file contains the cleaned and structured results of the **Metropolitano 2026 Semifinals** (all dance categories).  
It was generated by two scripts:  
1. `main.py` → extracts raw tables from `metrosemi.pdf` into a flat `col_1`/`col_2`/… format  
2. `post_process.py` → transforms that flat data into the clean, grouped schema below.

---

### Overall Structure

```json
[
  {
    "category": "SEMIFINAL VALS",
    "scores": [ ...array of couples... ]
  },
  {
    "category": "SEMIFINAL MILONGA",
    "scores": [ ... ]
  },
  {
    "category": "TANGO DE PISTA ADULT",
    "scores": [ ... ]
  },
  // ... one object per competition category
]
```

- **Root**: A JSON **array** (list) of category objects.
- Each category object has exactly **two keys**: `"category"` and `"scores"`.

---

### Detailed Field Descriptions

#### 1. Category Object

| Field      | Type     | Description                                      | Example                          |
|------------|----------|--------------------------------------------------|----------------------------------|
| `category` | string   | Official name of the competition category        | `"SEMIFINAL VALS"`               |
| `scores`   | array    | List of all participating couples in this category | `[ {…}, {…}, … ]`               |

#### 2. Couple Entry (inside `scores` array)

| Field       | Type          | Description                                                                 | Example Value                          |
|-------------|---------------|-----------------------------------------------------------------------------|----------------------------------------|
| `Nº`        | number (int)  | Couple’s registration number                                                | `57`                                   |
| `Nombres`   | string        | Names of the two dancers (format usually “Lastname1 - Lastname2”)          | `"Colodrero - Cartagena"`              |
| `scores`    | object        | Judge scores. **Keys** = judge names, **Values** = score (float or null)   | `{ "SAMETBAND": 8.8, "MARTINEZ": 8.5, ... }` |
| `PUNTAJE`   | number (float)| Total points (sum of all judge scores)                                      | `35.0`                                 |
| `PROMEDIO`  | number (float)| Average score per judge                                                     | `8.75`                                 |

**Important notes about `scores` object**:
- Judge names are taken directly from the PDF header row (e.g. `SAMETBAND`, `MARTINEZ`, `MISSÉ`, `GALLARDO`, `COLETTI`, `CEJAS`).
- A score can be:
  - `null` → the judge marked **“Abstencion”** (abstention)
  - `float` → normal score (always uses dot as decimal separator)
- The number of judges is **always 6** in this PDF (col_3 to col_8).

### Data Types Summary (for typing / validation)

```typescript
type ScoreEntry = {
  Nº: number;
  Nombres: string;
  scores: Record<string, number | null>;   // judgeName → score
  PUNTAJE: number | null;
  PROMEDIO: number | null;
};

type CategoryResult = {
  category: string;
  scores: ScoreEntry[];
};

type MetropolitanoSemifinalData = CategoryResult[];
```

### How to Load in Python (quick reference)

```python
import json

with open("metropolitano_semifinal_clean.json", "r", encoding="utf-8") as f:
    data: list[dict] = json.load(f)

# Example: get all Vals couples
vals = next(cat for cat in data if cat["category"] == "SEMIFINAL VALS")
print(len(vals["scores"]))          # number of couples
print(vals["scores"][0]["Nombres"]) # first couple name
```
